# شات خدمة العملاء — One Trip (دليل المالك)

نظام دردشة خدمة عملاء متكامل بهوية One Trip، مربوط بلوحة التحكم، مع خدمات سريعة، وجاهز لربط واتساب بيزنس والتزامن السحابي عبر Supabase.

## نظرة عامة
- **ودجة عائمة** على كل صفحات الموقع (زر أسفل اليسار) — شكل شركات الدعم الكبرى، RTL، بألوان الهوية.
- **خدمات سريعة** (مش مجرد محادثة): أسعار السيارات، الفروع، احجز الآن، باقات شهرية، حلول الشركات، حالة طلبي، التحدث مع موظف.
- **رد آلي ذكي** بالكلمات المفتاحية + تصعيد لموظف بشري عند الحاجة.
- **صفحة محادثات في لوحة التحكم** (Inbox بنمط WhatsApp-Web/Intercom) للرد على العملاء مباشرة.
- **ربط واتساب بيزنس** (Meta Cloud API) عبر جسر خلفي — الموظف يرد من واتساب والرد يوصل لشاتنا (يحتاج نشر السيرفر).
- **تزامن عبر الأجهزة** عبر Supabase (اختياري) — بدل ما التزامن يبقى على نفس المتصفح بس.

## الملفات ودورها
| الملف | الدور |
|---|---|
| `chat.js` | نواة الشات — طبقة البيانات + الخدمات السريعة + الرد الآلي + جسر واتساب (client). مصدر الحقيقة. |
| `chat-widget.js` / `chat-widget.css` | ودجة الموقع العائمة. |
| `admin.html` (أقسام **المحادثات** / **واتساب بيزنس** / **مزامنة Supabase**) | لوحة التحكم: Inbox + الإعدادات. |
| `supabase-sync.js` | طبقة المزامنة السحابية الاختيارية (تشتغل فقط لو ضبطت Supabase). |
| `supabase/schema.sql` + `supabase/README.md` | جداول Supabase + دليل الإعداد. |
| `whatsapp/` | جسر واتساب الخلفي + ملفات النشر + دليل. |
| `CHAT_CONTRACT.md` | العقد التقني (مرجع المطوّرين — لا يُغيَّر بدون مزامنة كل الأجزاء). |

## إزاي بيشتغل دلوقتي (بدون أي إعداد)
كل الشات يشتغل فورًا على `localStorage`:
- الزائر يبدأ محادثة من الودجة → تظهر في **لوحة التحكم → المحادثات** → الموظف يرد → الرد يوصل للزائر.
- التحديث **لحظي** عبر `BroadcastChannel` بين تبويبات/صفحات نفس المتصفح.
- القيد: التزامن على **نفس المتصفح/الجهاز فقط**. للتزامن عبر الأجهزة فعّل Supabase (تحت).

## كيف تجرّب الآن
1. افتح `index.html` و `admin.html` في تبويبين (كلمة مرور اللوحة: `onetrip`).
2. من الموقع: افتح الودجة، اضغط خدمة سريعة (مثلًا "أسعار السيارات") أو اكتب رسالة.
3. روح لتبويب اللوحة → **المحادثات** → هتلاقي المحادثة → ردّ عليها.
4. ارجع لتبويب الموقع → الرد ظهر في الودجة لحظيًا.

## تفعيل التزامن عبر الأجهزة (Supabase)
1. أنشئ مشروع Supabase ونفّذ `supabase/schema.sql` (SQL Editor). التفاصيل في `supabase/README.md`.
2. لوحة التحكم → **مزامنة Supabase** → ضع Project URL + anon key → فعّل → احفظ → أعد التحميل.
3. **تنويه أمني:** سياسات الديمو مفتوحة للتجربة — قيّدها قبل الإنتاج (موضّح في `supabase/README.md`).

## تفعيل واتساب بيزنس
ربط واتساب الحقيقي يحتاج **سيرفر HTTPS عام** (مش ممكن من موقع ثابت لوحده). الجسر جاهز في `whatsapp/`:
1. انشر `whatsapp/` (Render / Railway / Docker / VPS) — راجع `whatsapp/README.md`.
2. اضبط في Meta: Webhook = `https://YOUR-DOMAIN/whatsapp/webhook` + verify token.
3. لوحة التحكم → **واتساب بيزنس** → املأ Phone Number ID / Token / Verify Token.
4. (موصى به) اضبط `SUPABASE_URL` + `SUPABASE_SERVICE_KEY` على السيرفر → رسائل واتساب الواردة تظهر **لحظيًا** في الودجة واللوحة عبر Supabase Realtime.

> بدون نشر السيرفر، رد الموظف من تطبيق واتساب لسه غير متصل — لكن شات الموقع يشتغل تمام لوحده.

## الطريق للإنتاج
نفس الأشكال (conversations / messages) تنتقل كما هي لـSupabase + نشر جسر واتساب → دردشة موحّدة عبر كل القنوات والأجهزة. (يطابق ملاحظات المشروع: تجريبي على localStorage → Supabase لاحقًا.)
