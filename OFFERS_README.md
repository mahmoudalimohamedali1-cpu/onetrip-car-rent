# One Trip — نظام العروض (Offers)

نظام عروض/خصومات لموقع تأجير السيارات «One Trip». الأدمن ينشئ عرضًا ويختار له سيارات،
فيظهر تلقائيًا على الموقع: **ريبون/شارة فوق صورة السيارة** + **سعر مخفّض** + سكشن **«عروضنا»** في الرئيسية.

## ماذا يفعل النظام؟
- **الأدمن** (داشبورد `admin.html` ← قسم «العروض»): ينشئ عرضًا (عنوان، خصم نسبة أو مبلغ، أيقونة، لون،
  تفعيل، تاريخ بداية/نهاية اختياري) ويختار له سيارة أو أكثر من السيارات المتاحة.
- **الموقع العام**: لكل سيارة عليها عرض فعّال:
  - **ريبون** فوق صورة السيارة يحمل عنوان العرض وأيقونته ولونه.
  - **السعر القديم مشطوبًا والجديد** بعد تطبيق الخصم.
  - السيارة تظهر داخل سكشن **«عروضنا»** في الصفحة الرئيسية.

## الملفات
| الملف | الدور |
|---|---|
| `offers-core.js` | قلب النظام. يوفّر `window.OneTrip.Offers`، يحقن سكشن «عروضنا» وريبونات الكروت تلقائيًا، ويحقن الـCSS الخاص به (ببادئة `.ot-offers`). |
| `admin.html` (قسم «العروض») | واجهة الأدمن لإنشاء/تعديل/حذف العروض. |
| `index.html` / `fleet.html` / `create-booking.html` | تستدعي `offers-core.js` (بعد `cars.js`) ليعمل الحقن التلقائي. |

> يجب تحميل `cars.js` **قبل** `offers-core.js`. الحقن في الصفحات تلقائي — لا حاجة لتعديل HTML الصفحات
> أكثر من إضافة وسم السكربت.

## الربط بالسيارات
الربط يتم **بمعرّف السيارة (car id)**. يخزّن العرض مصفوفة `carIds` (مثل `['camry23','accent25']`)،
ويطابقها `offers-core.js` مع السيارات من `OneTrip.bookingCars()` لعرض الريبون والسعر المخفّض على السيارة الصحيحة.

## نموذج البيانات
يُخزَّن في `localStorage` تحت المفتاح **`ot_offers`** (مصفوفة عروض). شكل العرض:

```js
{
  id:'offer_<ts>',
  title:'عروض عودة المدارس',         // يظهر في الريبون
  carIds:['camry23','accent25'],     // معرّفات السيارات المشمولة
  discountType:'percent'|'amount',   // نوع الخصم
  discountValue:20,                  // القيمة (٪ أو ريال)
  icon:'school',                     // مفتاح أيقونة: school/national/tag/flash/snow/sun/percent/gift/star/fire
  color:'#e0322b',                   // لون الشارة
  active:true,
  startsAt:'2026-08-01', endsAt:'2026-09-15',  // اختياري (فاضي = بلا حد)
  order:0
}
```

أهم دوال `window.OneTrip.Offers`: `offers()`، `saveOffer(o)`، `deleteOffer(id)`، `activeOffers()`،
`offerForCar(carId)`، `discounted(carId, basePrice)`، `badgeHTML(offer)`، `mountHomeOffers()`.
كل عمليات `localStorage` داخل `try/catch` (لا ترمي أخطاء)، وكل تعديل يبثّ حدث `change` عبر
`BroadcastChannel('ot_offers')` لمزامنة التبويبات المفتوحة.

## ملاحظة للإنتاج (Supabase)
التخزين الحالي عبر `localStorage` **تجريبي** (لكل متصفح على حدة). للإنتاج: انقل `ot_offers` إلى جدول
`offers` في **Supabase**، واستبدل دوال القراءة/الكتابة في `offers-core.js` باستعلامات Supabase
(مع Realtime بدل `BroadcastChannel` للمزامنة)، وطبّق سياسات RLS بحيث القراءة عامة والكتابة للأدمن فقط.
