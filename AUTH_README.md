# One Trip — نظام حسابات العملاء (Customer Accounts)

نظام تسجيل دخول / حساب جديد / ملف شخصي **لزوّار الموقع (العملاء)** — مختلف تمامًا عن
حسابات موظفي لوحة التحكم (`ot_users`). مبني بنفس أسلوب المشروع: Vanilla JS، IIFE، RTL،
namespace `window.OneTrip`، وطبقة بيانات على `localStorage` (تجريبي → Supabase Auth لاحقًا).

---

## الفكرة

- يقدر زائر الموقع ينشئ حساب، يسجّل دخول، ويدير ملفه الشخصي.
- بمجرد تسجيل الدخول تظهر بياناته في الترويسة، وتتعبّأ تلقائيًا في فورمات الحجز/التواصل.
- العميل يقدر يشوف **حجوزاته** ومحادثاته من صفحة البروفايل، والأدمن يشوف كل العملاء من اللوحة.

---

## الملفات

| الملف | الوصف |
|---|---|
| `auth-core.js` | منطق الحسابات: `window.OneTrip.Auth` (signup/login/logout/current/update/changePassword/deleteAccount + bookings/conversations/leads). البيانات في `localStorage`. |
| `auth-widget.js` + `auth-widget.css` | الودجة في الترويسة: أزرار «تسجيل الدخول»/«حساب جديد» + مودال، وعند الدخول اسم العميل + قائمة (ملفي/حجوزاتي/خروج). ستايل ببادئة `.otauth-`. |
| `profile.html` | صفحة البروفايل: تعديل البيانات، تغيير كلمة المرور، قسم «حجوزاتي» (`#bookings`)، محادثاتي، خروج، حذف الحساب. |
| `admin.html` (قسم «العملاء») | جدول كل العملاء من `ot_customers` (الاسم/الإيميل/الجوال/المدينة/عدد الحجوزات/تاريخ التسجيل) + بحث + تفاصيل. للأدمن فقط. |
| `auth-prefill.js` | glue صغير وآمن: لو العميل مسجّل دخول، يملأ تلقائيًا حقول الاسم/الإيميل/الجوال الفاضية في الفورمات. لا يكتب فوق قيمة موجودة، ولا يرمي أخطاء. |

### مفاتيح التخزين (localStorage)

| المفتاح | الوصف |
|---|---|
| `ot_customers` | مصفوفة حسابات العملاء |
| `ot_customer_session` | معرّف العميل المسجّل حاليًا |

نموذج العميل: `{ id, name, email, phone, city, pass:'<hash>', createdAt, updatedAt, meta }`
(الإيميل فريد ويُخزَّن lowercased؛ كلمة المرور تُخزَّن كهاش بسيط — تجريبي).

---

## التدفّق (Flow)

1. **حساب جديد (Signup):** من ودجة الترويسة → مودال → تبويب «حساب جديد».
   التحقق: إيميل صحيح، جوال ≥ 9 أرقام، كلمة مرور ≥ 6. الإيميل لازم يكون فريد.
2. **تسجيل الدخول (Login):** بالإيميل **أو** الجوال (يطابق آخر 9 أرقام) + كلمة المرور.
3. **بعد الدخول:** الودجة تعرض اسم العميل وقائمة سريعة، وكل الصفحات تتحدّث فورًا عبر
   `Auth.on('change')` (BroadcastChannel `ot_auth` + حدث `storage`).
4. **البروفايل (`profile.html`):** تعديل البيانات (`Auth.update`)، تغيير كلمة المرور
   (`Auth.changePassword`)، عرض الحجوزات، تسجيل الخروج، وحذف الحساب (`Auth.deleteAccount`).

---

## الربط ببقية الموقع

- **الحجوزات:** `Auth.bookings()` يقرأ `window.OTB.bookings()` ويُرشّحها بمطابقة الجوال
  (آخر 9 أرقام) أو الإيميل — قراءة فقط، لا يعدّل نظام الحجوزات. تظهر في البروفايل
  وفي عدد حجوزات العميل بلوحة الأدمن.
- **الشات:** `Auth.conversations()` يقرأ محادثات `OneTrip.Chat` المطابقة بالجوال (قراءة فقط).
- **التعبئة التلقائية:** `auth-prefill.js` يملأ حقول فورمات الحجز/التواصل لو فاضية.
- **لوحة التحكم:** قسم «العملاء» في `admin.html` يعرض `ot_customers` (للأدمن فقط).

### ترتيب تحميل السكربتات في الصفحات العامة

```
cars.js → auth-core.js → branch-map.js → chat.js → supabase-sync.js → chat-widget.js → auth-widget.js → auth-prefill.js
```

الصفحات العامة الموصولة: `index, fleet, create-booking, long-term, corporate, contact,
about, terms, rental-terms, privacy, cancellation, cookies`. الإضافة idempotent (لا تتكرّر).

---

## ملاحظة أمنية (DEMO فقط)

> كلمات المرور تُخزَّن **محليًا في المتصفح** بهاش بسيط **غير آمن** — للتجربة فقط.
> لا تخزّن أسرارًا حقيقية. الإنتاج لازم ينتقل إلى **Supabase Auth** (تسجيل/دخول
> آمن على الخادم، تخزين مُملّح/مُهاش للكلمات، جلسات JWT)، مع نقل طبقة `Auth`
> لتنادي Supabase بدل `localStorage` — بدون تغيير واجهة `window.OneTrip.Auth` المستخدمة
> في باقي الموقع.

## كيف تختبر

1. افتح `index.html` في المتصفح (أو أي صفحة عامة) — هيظهر زرّا «تسجيل الدخول»/«حساب جديد» في الترويسة.
2. أنشئ حساب جديد (إيميل + جوال + كلمة مرور ≥ 6) — لازم يسجّل دخول مباشرة ويظهر اسمك.
3. افتح `create-booking.html` أو `contact.html` — لازم تتعبّأ حقول الاسم/الإيميل/الجوال تلقائيًا (لو فاضية).
4. سجّل خروج ثم ادخل بالجوال بدل الإيميل — لازم ينجح.
5. افتح `profile.html` — عدّل بياناتك، غيّر كلمة المرور، وراجع قسم «حجوزاتي».
6. افتح `admin.html` بحساب أدمن → قسم «العملاء» — لازم يظهر الحساب الجديد وعدد حجوزاته.
7. تحقق سريع من `localStorage`: المفتاحان `ot_customers` و`ot_customer_session` موجودان.
